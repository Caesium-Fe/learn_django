  6. Limit和Target值

    1. 当缺失limit时，且UI->Distribution->Limit上选择Ingore或Count时

      若station type在('cell', 'omnia txrx', 'omnia cell', 'lat', 'uat', 'zat', 'omnia system', 'zephyr cell')中

      首先将测试项中小于-200或者大于200的测试值剔除，只保留[-200, 200]的有效测试值(有效指的是可以转为float类型)

      注：后续的所有指标也都是使用剔除之后的数据计算---所以这玩意儿不是应该在一开始的时候就做好处理吗？为何要等到计算时才判断？

      所以如果limit不缺失时且station type又在上面中的一个时，会不会做这个剔除的操作呢？

       

      target = avg(测试项的有效测试值)

      std = std(测试项的有效测试值)

      upperlimit = target + limit_spec * std

      lowlimit = target - limit_spec * std

      其中limit_spec = UI->Distribution->SPEC/前面的数

       

    2. 当缺失limit时，且UI->Distribution->Limit上选择Delete时

      不计算，因为这个测试项是不会写入的

       

    3. limit完整时

      upperlimit和lowlimit从report中读出

      计算逻辑ByconfigData.py -> _getItemTarget

      1. 当item中包含“evm”时

        target = 20 * numpy.log10 ( ( 10 ** (upperlimit / 20.) + 10 ** (lowlimit / 20.) ) / 2 )

      2. 否则

        target = (upperlimit + lowlimit) / 2

  7. Max、Min、CPK、Stdev、Mean、Shifts指标

    这些指标在计算之前可能要先做一次测试项的测试值筛选的逻辑：与UI->ALL->Delete_limit有关

    xxxxxxxxxx
    # 筛选逻辑：
    values = [val for val in raw_values if lowlimit - delete_spec <= val <= upperlimit + delete_spec ]
    """
    其中：
    raw_values: 测试项的原测试值
    delete_spec: 
        1.当station type == 'sota'时：
            delete_spec = (upperlimit - lowlimit) / percent * ((1-percent)/2)   注：percent UI->ALL->Delete_limit */* 后面那个数
        2.否则
            delete_spec = UI->ALL->Delete_limit */* 前面那个数
            
    upperlimit: Limit 和 Limit2(如果存在的话)中最大的值
    lowlimit: Limit 和 Limit2(如果存在的话)中最小的值
    """

    1. 当UI->ALL->Delete Data Type选择Auto时

      1. 当len(values) / len(raw_values) > 0.7时

        使用筛选逻辑

      2. 否则

        不使用筛选逻辑，保留所有有效的测试项的测试值(raw_values)

    2. 当UI->ALL->Delete Data Type选择ALL时

      使用筛选逻辑

    3. 当UI->ALL->Delete Data Type选择Keep时

      不使用筛选逻辑，保留所有有效的测试项的测试值(raw_values)
